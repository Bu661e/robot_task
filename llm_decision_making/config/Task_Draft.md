我目前在做一个机械臂抓取系统。

在我的设想中，我使用 Isaac Sim 搭建仿真环境，机械臂采用 Franka 机械臂。桌面上会摆放一些 YCB 数据集中的物体，主要包括圆罐、方块、盒子等易于抓取的物体，以及一些简单的几何体。

在桌面正上方，会安装一个俯视摄像头。

整个系统主要分为三个部分：

1. 感知部分：基于 SAM3 和 3D 感知模型。
2. 大模型决策部分：基于千问 3.5。
3. 执行部分：基于 Franka 的 `PickPlaceController`。

接下来，我会以一个具体任务为例进行说明。

当前任务是：拿起桌面上最高的瓶子。

首先，系统会将整个任务发送给大模型，由大模型对任务进行拆解，提取其中涉及的关键物体。以本次任务为例，大模型会提取出“瓶子”这一目标物体。

随后，位于上方的摄像头会拍摄一张桌面图像，并进一步计算得到对应的 point map。

接着，系统会将“瓶子”这个文本提示词和摄像头拍摄的图像一起发送给 SAM3 模型。SAM3 会根据输入图像和提示词生成分割结果。由于桌面上可能存在多个瓶子，因此模型会输出多个候选 mask。

之后，系统会将图像、mask 以及 point map 一并输入到 SAM 3D 模型中。经过推理后，SAM 3D 会为每个 mask 对应的物体输出其基础信息，包括 `6drotation_normalized`、`scale`、`translation`、`rotation`，以及高斯表示（GS）等信息。

在当前阶段，我们暂时只使用物体的基础信息，例如 `scale`、`translation` 和 `rotation`，暂不使用高斯表示数据。

然后，我会将机械臂可调用的 API、感知模块得到的桌面物体位姿信息，以及当前任务描述，一并提供给千问模型，让它以代码的形式生成执行流程。这一部分采用的是 Code as Policies 的思想，详细内容可以参考 `prompt-EN.yaml`。

最后，系统会将大模型输出的内容视为一段新的代码，并通过 Python 执行，从而完成整个抓取任务。
