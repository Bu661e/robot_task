# AGENTS

## 仓库级协作约束

- 开始开发前，先确认自己当前在哪个分支，使用 `git branch --show-current` 核对，不要在错误分支上直接开发。
- 开始开发前，先确认自己当前在哪个 worktree 和工作目录，必要时使用 `pwd` 与 `git worktree list` 核对，避免在错误 worktree 中修改代码。
- 开发时必须注意模块边界；如果当前任务只属于某个模块，就只修改该模块和必要的共享文档。
- 没有明确需求时，不要顺手修改其他模块代码，不要把当前模块的逻辑扩散到其他模块里。
- 如果确实需要跨模块改动，先明确接口边界、影响范围和改动原因，再继续实施。
